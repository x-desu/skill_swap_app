# SkillSwap Codebase & Infrastructure Overview

This document summarizes the internal configuration, extra tools, environments, and special deployment details based on the current state of the codebase.

## 1. Extra Tools & Services
A review of the `package.json` and `app.json` reveals the following services and tools configured in the application:

*   **Payments & Monetization**: 
    *   **Razorpay**: Integrated directly (`react-native-razorpay`) and currently seeded with a production key (`rzp_live_SX039qZ9OOIngV`) in `app.json`.
    *   **RevenueCat**: Available in dependencies (`react-native-purchases`), indicating a hybrid setup handling both direct micro-transactions and subscription modeling.
*   **Authentication Identity Providers**: Native integration for **Google Sign-In** (`@react-native-google-signin/google-signin`) and **Apple Authentication** (`@invertase/react-native-apple-authentication`).
*   **Video & Real-Time Communication**: Utilizes **Stream Video SDK** (`@stream-io/video-react-native-sdk`) alongside WebRTC (`@stream-io/react-native-webrtc`) for 1-on-1 interactions.
*   **State Management**: Both **Redux Toolkit** and **Zustand** are currently listed, suggesting a potential migration or split-state logic across different modules.
*   **Location & Notifications**: Leverages `expo-location` and `react-native-maps` for geospatial features, and `expo-notifications` for background alerts.
*   *(Note: No explicit crash-reporting tools like Firebase Crashlytics or remote feature flag services like Firebase Remote Config are currently installed.)*

## 2. Environments (Dev, Staging, Prod)
Currently, **there is no formal environment separation**. 
*   The project uses a static `app.json` rather than a dynamic `app.config.js`. 
*   Variables and configurations (like the Razorpay Live key and the Firebase bundle configuration for `com.skillswap.app`) are hardcoded into `app.json`.
*   To support multiple environments in the future (e.g., separating bleeding-edge test data from production data), the project will need to migrate to `app.config.js` and use environment variables (e.g., `process.env.APP_ENV`).

## 3. Special Deployment & Build Details
The way this project is built has specific low-level iOS and Expo modifications:

*   **Expo Prebuild / CNG (Continuous Native Generation)**: The project heavily uses Expo Config Plugins. Because it relies heavily on custom native modules (WebRTC, Razorpay, specialized Firebase), it cannot run in the standard Expo Go app. It must be built locally using `npx expo run:ios` or `npx expo run:android`.
*   **Firebase Integration**:
    *   Firebase is configured at the native level via `GoogleService-Info.plist` for iOS and `google-services.json` for Android.
    *   **CRITICAL iOS CONFIG**: To prevent compatibility collisions with WebRTC, the `expo-build-properties` plugin is used to forcefully statically link Firebase (`useFrameworks: "static"` and `forceStaticLinking` on modules like `RNFBApp`, `RNFBAuth`, `RNFBFirestore`).
*   **EAS (Expo Application Services)**: 
    *   There is no `eas.json` file in the root. This indicates the application is not configured for cloud builds via EAS Build. Builds are currently generated by compiling locally. To utilize cloud CI/CD, EAS will need to be initialized. 
*   **UI Architecture**: Relies on NativeWind (`nativewind`, `tailwindcss`) patched with `react-native-css-interop` alongside some Gluestack UI utilities, configured heavily inside `babel.config.js`.
